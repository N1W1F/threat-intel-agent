# تسليم واجب المعسكر — نظام وكلاء الذكاء الاصطناعي

> **الاسم:** Threat Intel & Asset Monitor
> **النوع:** نظام متعدد الوكلاء (Multi-Agent System) — تطبيق سطح مكتب مستقل (Windows)
> **الشق المطبّق:** الشق الثاني — Golden Dataset لاختبار أمان الوكلاء (126 اختبار حقيقي)
> **المستودع:** [github.com/N1W1F/threat-intel-agent](https://github.com/N1W1F/threat-intel-agent) (خاص) — مع CI/CD تلقائي

---

## 1. فكرة المشروع

نظام أمني متعدد الوكلاء يؤتمت رصد ثغرات البرامج المثبتة على الجهاز ويقرر أيها يستحق اهتمامك الآن فعلاً. بدل المراجعة اليدوية لعشرات البرامج، مجموعة وكلاء متخصصة تتعاون تحت وكيل مدير (Orchestrator) لإنجاز المهمة تلقائياً: فحص → مطابقة ثغرات → تصنيف قرار → تقرير.

**المشكلة المحلولة:** متابعة الثغرات يدوياً بطيئة، والتنبيهات الخام (كل تطابق نصي بقاعدة CVE) مليئة بالضجيج — النظام يفرز العاجل الحقيقي من الضجيج تلقائياً.

---

## 2. الهيكلة (Architecture)

```
                    👤 المستخدم
                        │
                🧭 Orchestrator (الوكيل المدير)
                        │
     ┌──────────┬───────┼───────┬──────────┬─────────────┐
     ▼          ▼       ▼       ▼          ▼             ▼
🗂️ Asset   🛰️ Threat  🚨 KEV  ⚖️ Decision  📝 Remedi-  🔄 Package
 Auditor    Hunter    Checker   Agent       ation       Manager
(قراءة     (شبكة     (قائمة   (تصنيف،     (يكتب       (يفحص/يطبّق
 فقط)       فقط)      CISA)    بلا تنفيذ)   التقرير)    تحديثات winget)
     │          │       │       │           │             │
inventory.txt  NVD    CISA    findings.json report.md   winget upgrade

         + 🤖 Analyst (LLM محلي عبر Ollama — شرح/سؤال وجواب/إعادة تقييم)
```

| الوكيل | الدور | الصلاحية (أقل امتياز) |
|---|---|---|
| Orchestrator | تنسيق التتابع وجمع النتائج | تنسيق فقط |
| Asset Auditor | قراءة البرامج المثبتة عبر winget | قراءة فقط، داخل الصندوق |
| Threat Hunter | بحث CVE في NVD + كاش محلي | شبكة فقط، مضيف NVD ثابت |
| KEV Checker | مطابقة CISA KEV (ثغرات مستغلة فعلياً بالعالم الحقيقي) | شبكة فقط، مضيف CISA ثابت |
| Decision Agent | تصنيف عاجل/روتيني/معلوماتي + مؤشر صحة | **صفر صلاحية تنفيذ — لا يستورد subprocess إطلاقاً (مُتحقَّق بـ AST)** |
| Remediation | كتابة التقرير النهائي | كتابة مسار واحد ثابت |
| Package Manager | فحص/تطبيق تحديثات winget | argv ثابت، بلا shell، يرفض أي معرّف غير مفحوص |
| Analyst | شرح/سؤال وجواب/إعادة تقييم عبر LLM محلي | لا صلاحية تنفيذ، حماية Prompt Injection على بيانات CVE |

## 3. الوظائف

- فحص أمني متعدد الوكلاء ينتج تقريراً حقيقياً بثغرات مطابقة لإصدارات الجهاز الفعلية (لا بيانات وهمية بأي مكان — مُتحقَّق بثلاث مراجعات كود مستقلة).
- **مؤشر صحة أمنية** (0-100) محسوب حسب الخطورة **القابلة للتصرف فعلياً** (لها تحديث + استغلالها واقعي)، لا حسب عدد التطابقات الخام — يتحرك فعلياً بين الفحوصات كلما أُصلح شيء.
- **مطابقة CISA KEV**: يميّز "خطر نظري" عن "ثغرة مستغلة فعلياً بالعالم الحقيقي".
- فحص وتطبيق تحديثات البرامج تلقائياً عبر winget، مع استبعاد الألعاب وحزم إعادة التوزيع من ضجيج فحص NVD.
- **تاريخ المخاطر**: رسم بياني تفاعلي (منحنى ناعم + تدرّج) — كل نقطة قابلة للضغط لعرض تفاصيل ذلك الفحص (التوزيع، التغيّر عن السابق).
- **إشعارات سطح مكتب حقيقية** (Windows Toast) عند اكتشاف عنصر عاجل بفحص دوري تلقائي — لا بريد إلكتروني (أُزيل بالكامل لتبسيط سطح الهجوم؛ الإشعار المحلي أكثر أماناً من بيانات SMTP مخزّنة).
- واجهة ثنائية اللغة (عربي/إنجليزي) كاملة RTL/LTR، تصميم SOC-dashboard داكن.
- **تصميم ثلاثي الأبعاد تفاعلي** (Three.js محلي بالكامل — بدون CDN): خلفية شبكة جزيئات متحركة تتلوّن حسب وضعك الأمني الحقيقي، كرة صحة 3D، بطاقات KPI تميل تفاعلياً نحو الماوس.
- تطبيق سطح مكتب مستقل (`ThreatIntelAgent.exe`) + حزمة مشاركة آمنة (كود مصدري + SHA-256).
- **CI/CD** عبر GitHub Actions: كل push/PR يشغّل الـ126 اختبار تلقائياً، ويبني الـexe فقط لو نجحت، مع Branch Protection يمنع الدمج لـmain بدون اجتياز الاختبارات.

## 5. ميزات الذكاء الاصطناعي والأتمتة

- **شرح ذكي (LLM)**: زر "اشرح" بجانب كل ثغرة يشرحها بعربي مبسّط لمستخدم غير تقني.
- **إعادة تقييم ذكية (LLM)**: وكيل يفلتر نتائج NVD ويحدد أيها فعلاً ذو صلة — قرار agentic حقيقي.
- **سؤال وجواب طبيعي (LLM)**: تسأل عن تقريرك بلغتك ويجاوب من بياناته.
- **مقارنة الفحوصات**: يعرض الثغرات الجديدة والتي انحلّت منذ آخر فحص.
- **فحص دوري تلقائي**: كل (3 أيام / أسبوع / أسبوعين / شهر)، مع إشعار Windows فوري عند وجود عنصر عاجل.

**النموذج**: محلي عبر **Ollama** (مجاني، بلا تكلفة، بلا إنترنت) — يطابق قاعدة المشروع "AI فعلي، صفر تكلفة".

### قيود الأمان لهذه الميزات
- بيانات CVE (غير موثوقة) تُغلَّف بكتلة معزولة داخل نداء LLM، ورسالة النظام تمنع تنفيذها كتعليمات — **تحقق فعلي**: اختُبر ضد 32 محاولة حقن حقيقية (OWASP LLM01، DAN jailbreak family، deepset/prompt-injections dataset) — 32/32 صمدت.
- الأسرار (مفتاح NVD الاختياري) مشفّرة DPAPI عند الحفظ، لا تظهر بأي رد API ولا سجل، ومستبعدة من نسخة المشاركة عبر `.gitignore`.
- كل نقاط API محميّة CSRF + Host allowlist، ونداءات LLM محدودة المعدّل.

## 6. وكيل القرار + مؤشر الصحة الأمنية + مطابقة الاستغلال الفعلي

- **KEV Checker Agent**: يقارن كل CVE بقائمة CISA الرسمية للثغرات المستغلة فعلياً بالعالم الحقيقي.
- **Decision Agent**: يصنّف كل نتيجة (عاجلة / روتينية / معلوماتية) بناءً على الخطورة + الاستغلال الفعلي + توفر تحديث + صعوبة الاستغلال + هل النسخة المُصلحة أصلاً أحدث من نسختك، ويحسب **مؤشر صحة أمنية** (0-100) من الخطر القابل للتصرف فعلياً فقط.
- **قيد الأمان الصارم**: وكيل القرار **لا يملك أي صلاحية تنفيذ** — لا يستورد `package_manager` ولا `subprocess` إطلاقاً (مُتحقَّق آلياً عبر تحليل AST بالكود، مو بس وعد). التنبيه العاجل بالواجهة زرّين فقط: **"اذهب للتحديث"** (بلا تنفيذ تلقائي) أو **"أجّل"** (بيانات توقيت فقط). القرار النهائي دائماً للمستخدم.

## 7. الواجهة — SOC Console بتصميم ثلاثي الأبعاد تفاعلي

واجهة كاملة من الصفر: شريط جانبي، شريط مؤشرات KPI (صحة/نتائج/حرجة/أصول) بلون ديناميكي حسب قيمتها الفعلية، لوحة تنبيهات عاجلة، تاريخ مخاطر تفاعلي قابل للضغط.

**الطبقة ثلاثية الأبعاد** (Three.js — مُحمَّل مرة واحدة وقت البناء، مُخزَّن محلياً بـ`app/static/vendor/`، **صفر اتصال إنترنت وقت التشغيل**، يطابق CSP الصارمة `script-src 'self'`):
- خلفية شبكة جزيئات متحركة تتلوّن فعلياً حسب مؤشر الصحة الحقيقي (أخضر/كهرماني/أحمر).
- كرة صحة 3D حقيقية (WebGL) بدل الحلقة المسطحة.
- بطاقات KPI بميلان 3D تفاعلي (CSS perspective) نحو مؤشر الماوس.

راجع [docs/PRD-3D-interactive-redesign.md](docs/PRD-3D-interactive-redesign.md) للتفاصيل الكاملة وقرار التصميم (لماذا مكتبة محلية موثّقة المصدر لا CDN خارجي).

## 8. CI/CD — من "يشتغل عندي" إلى "منتج حقيقي"

مطبّق كاملاً هذا الموسم:
- **Git**: مستودع خاص [github.com/N1W1F/threat-intel-agent](https://github.com/N1W1F/threat-intel-agent).
- **GitHub Actions** ([.github/workflows/ci.yml](.github/workflows/ci.yml)): كل push/PR على `main` يشغّل الـ126 اختبار على `windows-latest` (نفس بيئة التشغيل الحقيقية) + فحص استيراد كل موديول. لو نجحت، job ثانٍ يبني `ThreatIntelAgent.exe` تلقائياً ويرفعه كـartifact.
- **Branch Protection**: `main` محمي — أي دمج يتطلب نجاح check "Golden Dataset security test suite" أولاً، حتى للمشرفين.
